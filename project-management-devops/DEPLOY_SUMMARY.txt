QUICK START (LOCAL DEVELOPMENT)
===============================================================================

Prerequisites: Node.js 20+, Docker, bash

1. Install dependencies:
   $ make bootstrap

2. Start services (PostgreSQL, backend, frontend):
   $ bash scripts/compose.sh up

3. Test endpoints:
   - Frontend: http://localhost:80
   - Backend API: http://localhost:3000
   - Health check: http://localhost:3000/health

4. Register and login:
   $ curl -X POST http://localhost:3000/auth/register \
     -H "Content-Type: application/json" \
     -d '{"email":"user@example.com","password":"secret123","fullName":"John Doe"}'

5. Stop services:
   $ bash scripts/compose.sh down

For detailed commands, see QUICKSTART.md

PRODUCTION DEPLOYMENT PATH
===============================================================================

Phase 1: AWS Account Setup (2-3 hours)
   ☐ Configure OIDC provider for GitHub Actions
   ☐ Create ACM certificate for your domain
   ☐ Create ECR repositories
   ☐ Prepare or select base AMI (Ubuntu 22.04 LTS)

Phase 2: Configure Secrets (30 minutes)
   ☐ Add 10 GitHub Secrets (DB_PASSWORD, JWT_SECRET, etc.)
   ☐ Update infra/environments/prod.tfvars with actual values
   ☐ Enable branch protection on main branch

Phase 3: Deploy Infrastructure (15 minutes)
   $ cd infra/terraform
   $ terraform init           # Initialize with S3 backend
   $ terraform plan -var-file=../environments/prod.tfvars
   $ terraform apply -var-file=../environments/prod.tfvars

Phase 4: Verify Deployment (30 minutes)
   ☐ Check CloudFront distribution
   ☐ Access frontend via CloudFront domain
   ☐ Register test user
   ☐ Verify CloudWatch logs and metrics

Phase 5: Production Hardening (ongoing)
   ☐ Security audit checklist (see docs/security.md)
   ☐ Load testing and autoscaling verification
   ☐ Establish on-call rotation and runbooks
   ☐ Configure SNS alarms to Slack/email

See infra/environments/DEPLOYMENT.md for step-by-step guide.
